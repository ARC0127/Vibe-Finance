# Vibe Finance

Vibe Finance 是一个只研究中国大陆可获得信息、只执行虚拟交易的股票与基金决策实验。项目从 30,000 元虚拟本金开始，保存点时证据、决策、待成交订单、成交、费用与复盘；它不连接券商，也不承诺收益。

## 当前状态

截至 2026-07-20 午间，项目已有 20 只场内/场外基金研究工具，以及一个从沪深300/中证500成份股动态生成 30 只股票研究短名单的规则。研究池不是买入清单；当前行情、公司行为和历史门禁仍未全部通过，组合维持 NO_TRADE：

- 可投资现金：29,900 元
- DeepSeek 研究预算：100 元
- 股票/基金持仓：0 元
- 项目权益：30,000 元
- DeepSeek 实际调用：0 次，成本 0 元

现金不是遗漏的投资决定；在数据或风险门禁失败时，零仓位是系统允许的主动决策。

实际组合最多持有 6 个仓位、每轮最多新增 2 个买入，总权益不超过 60%，并限制核心权益、小盘成长、行业、红利、黄金、固收与现金管理等资产桶。同一指数的 ETF 和联接基金不能被当作两份分散。

## 一条命令

使用 Python 3.10+，无第三方运行依赖：

~~~bash
python3 -m vibe_finance run --input data/inbox/YYYY-MM-DD.json --mode short
~~~

常用检查：

~~~bash
python3 -m vibe_finance validate --input data/inbox/2026-07-19.json
python3 -m vibe_finance settle-open --input data/inbox/YYYY-MM-DD-open.json
python3 -m vibe_finance run-funds --input data/inbox/YYYY-MM-DD-funds.json
python3 -m vibe_finance status
python3 -m unittest discover -s tests -v
~~~

收盘分析固定为：校验点时输入 → 计算风险/趋势门禁 → 生成下一开盘虚拟订单 → 原子更新账本和心跳 → 写入不可覆盖的 Markdown/JSON 报告。下一交易日 09:35 的独立结算任务只处理已有订单，不生成新信号。

## 数据与成交纪律

- 每个数值必须带 as_of、来源标识和质量说明。
- 价格至少由 2 个来源交叉核验；策略输入至少积累 20 个历史点。
- 已知分红、拆分等公司行为必须写入 `corporate_actions`，并明确 `history_adjusted_for_corporate_actions`；存在公司行为但历史未复权时只允许 `WATCH/HOLD`，禁止生成订单。
- 收盘后信号只能在更晚的交易日开盘模拟成交，禁止同一收盘价回填成交。
- 开盘结算要求开盘价至少两个来源交叉核验；不满足时取消该虚拟订单。
- 费用模型计入双向券商佣金（暂按万三、单笔最低 5 元）、A 股双向过户费（0.001%）和 A 股卖出印花税（0.05%）；ETF 不收股票过户费或证券交易印花税。佣金是待按实际券商账户替换的模拟假设，交易所经手费视为已包含，避免重复计费。
- 休市、快照陈旧、市场冲击、价格冲突或历史不足都会 fail closed。
- 旧日报和决策 JSON 不可由常规运行覆盖；修正必须生成新记录。
- 专家观点只提供情景与反证，不直接触发 BUY/SELL。
- 天天基金是基金净值、申赎、费率、规模、经理和持仓的重要发现/交叉验证渠道；关键事实仍以基金公司、交易所或法定公告为准。
- 场外公募基金不能套用场内“下一开盘”成交模型；专用引擎按未知价原则登记 `PENDING_NEXT_NAV`，仅使用申请日之后公开并交叉核验的确认净值结算份额与申赎费。
- 目前没有稳定、许可明确的自动行情 API，因此每日任务负责用权威网页生成点时输入；无法核实时维持 NO_TRADE。

## 调度

- 开盘前复核：交易日 08:00，检查隔夜公告、公司行为和既有购买计划。
- 开盘成交结算：交易日 09:35，只结算上一收盘已生成的虚拟订单并记入全部费用。
- 短周期：交易日收盘后运行，生成下一可交易时点的虚拟决策。
- 基金净值周期：交易日 22:30，核验天天基金与基金公司公布的净值、申赎状态和费率，处理场外基金待确认订单。
- 长周期：每周运行，做组合归因、来源可靠性复核和规则版本审查。
- 反思进化：每周六 20:30，使用最近 5/20/60 个交易日做股票/基金分别归因和样本外策略审查。
- 文档日志整理：每天 23:10，增量索引本地文档、报告和日志，检查断链、重复与解析错误，不删除或覆盖历史文件。
- 活动监测：每 6 小时只读检查心跳和报告新鲜度。
- 高频周期：当前不创建、不启用。只有许可明确的点时数据、成交/滑点模型、走前回测、样本外通过和再次人工确认全部满足后才允许创建。

## DeepSeek 成本与密钥

用户声明 DeepSeek 余额为 100 元，本项目将其从可投资现金中隔离。未通过 API 核验余额，也未调用 API。每次真实调用都必须用 record-api-cost 记录模型、用途、输入/输出 token 和人民币成本，且累计不得超过 100 元。

密钥不得进入仓库、报告、日志或自动化提示。由于密钥曾出现在对话中，建议在 DeepSeek 控制台轮换后，仅通过本机环境变量注入。

## 风险边界

这是个人研究与代理能力训练项目，不是真实投资建议。模拟结果不能证明实盘收益能力；税费、流动性、涨跌停、停牌、基金申赎、净值发布时间、数据许可和行为偏差仍可能令实盘结果显著更差。
