# 定时设计

所有时间均为 Asia/Shanghai。平台自动化负责唤醒研究任务，Python 流水线负责确定性校验和账本更新。

## 开盘前复核

- 计划：周一至周五 08:00，为 09:30 开盘预留约 90 分钟研究缓冲。
- 核验交易日、隔夜公告、停复牌、公司行为和上一收盘购买计划。
- 只保留或取消已有计划，不使用当日未来数据回填，不直接记为成交。

## 开盘虚拟成交结算

- 计划：周一至周五 09:35。
- 只结算上一收盘已生成的待执行虚拟订单，不产生新信号。
- 输入使用当日开盘截点，`market_state` 必须为 `open` 或 `opening_auction_complete`，每个待成交标的必须提供至少两个 `open_source_ids`。
- 运行 `python3 -m vibe_finance settle-open --input data/inbox/YYYY-MM-DD-open.json`。
- 买入和卖出均计券商佣金；A 股双向计过户费、仅卖出计印花税；ETF 不计股票过户费或证券交易印花税。
- 佣金按账户可能不同，当前万三、最低 5 元仅为模拟假设；交易所经手费视为包含在佣金中，禁止重复计费。
- 开盘价缺失/冲突、超过买单限价、资金不足或持仓不足时取消订单并保留审计事件。

## 短周期

- 计划：周一至周五 16:30。
- 先核验交易所日历；非交易日只记录休市，不伪造行情或订单。
- 若当日不可变报告已经存在，读取并核验，禁止覆盖。
- 从 A/B 级来源优先采集，C 级只交叉验证。
- 创建 data/inbox/YYYY-MM-DD.json，先运行 validate，通过后运行 --mode short。
- 任何数据缺失、冲突、陈旧或市场冲击都 fail closed。
- 只生成虚拟的下一开盘订单，不连接真实交易。

## 长周期

- 计划：每周日 20:00。
- 汇总本周日报、订单、持仓、费用、来源失败和基准。
- 只在已结束样本上审查策略；样本不足时不得改规则。
- 运行 --mode long 或生成等价周报，已有同日长报时不得覆盖。

## 场外基金净值周期

- 计划：周一至周五 22:30，等待交易日净值通常完成发布后再运行。
- 同时核验天天基金与基金公司/法定披露；只有净值日期、申赎状态、规模和完整费率均通过门禁才进入决策。
- 运行 `python3 -m vibe_finance run-funds --input data/inbox/YYYY-MM-DD-funds.json`。
- 新申购按未知价原则登记为 `PENDING_NEXT_NAV`，不得用申请当日已知或回填净值立即成交；后续只使用更晚公开的确认净值结算份额和申赎费。
- 场外基金订单不由 09:35 开盘任务处理；基金净值缺失、陈旧或来源冲突时维持 WATCH。

## 反思改进与策略进化

- 计划：每周六 20:30。
- 分别汇总最近 5、20、60 个交易日的股票、场内基金、场外基金、现金/债券/黄金表现，再做组合归因。
- 检查收益、回撤、波动、换手、费用、成交偏差、取消原因、来源失败、公司行为、净值时间、风格漂移和资产桶集中度。
- 每个候选改进必须有预注册假设、训练/验证/样本外区间、成本压力测试、接受阈值、反证和回滚版本。
- 策略参数至少需要 20 笔已完成虚拟交易并通过走前与样本外门禁才可升级；样本不足只登记 PROPOSED_ONLY。
- 任何升级都要提升策略版本、保留 rollback_base 并通过全部测试，禁止因单周盈亏或基金排行榜直接调参。

## 本地文档与日志整理

- 计划：每天 23:10，在 22:30 基金净值任务之后运行。
- 增量清点规范文档、配置、研究数据、决策/结算/进化报告和审计日志，记录路径、大小、修改时间、SHA-256 与解析状态。
- 只读校验 JSON/JSONL，检查 Markdown 本地链接、命名/日期一致性、重复内容和孤立文件。
- 生成不可覆盖的 `reports/document-log/YYYY-MM-DD-HHMM.md` 和对应 JSON，并原子维护 `docs/DOCUMENT_LOG_INDEX.md`。
- 禁止自动删除、移动、改名、压缩或提交用户文件；归档和去重只提供候选清单，等待用户确认。
- 不读取或回显密钥，不调用付费 API，不修改账本、订单、策略或金融流水线心跳。

## 活动监测

- 计划：每 6 小时。
- 运行 python3 -m vibe_finance status。
- 只读检查 data/ledger/heartbeat.json、最近决策报告和 API 预算。
- 监测任务本身禁止更新心跳、订单或账本。
- 常规阈值为 36 小时；考虑周末/法定休市时，以最近应运行的短周期和 8 天内的长周期为补充判断。
- 只有状态陈旧、文件缺失、预算异常或运行失败时才需要告警。

## 高频状态

当前为 DISABLED_NOT_CREATED。只有以下条件全部通过才允许创建：

1. 许可明确的逐笔/盘口点时数据；
2. 正确交易日历、公司行为和代码生命周期；
3. 排队、部分成交、涨跌停、停牌、滑点和费用模型；
4. 走前回测以及未用于调参的样本外测试；
5. 压力测试、资源与 API 成本预算；
6. 明确的启停、熔断和回滚规则；
7. 用户再次确认。

“高频定时”不是高频交易能力；没有上述条件时增加调用频率只会放大噪声和成本。
