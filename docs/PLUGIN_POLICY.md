# 插件使用政策

插件只能改善工作流，不得替代点时数据、策略配置、订单日志或虚拟账本。是否接入以“提高决策质量、审计能力或人类可读性”为标准，不因插件名称与金融相关就自动启用。

## Investment Banking

状态：**不接入 Vibe Finance 的股票/基金决策与定时任务**。

该插件面向并购、融资、资本市场发行、杠杆融资、重组、交易尽调、买方名单、路演和投行估值等 banker-owned 工作流；其官方使用边界明确排除公开股票投资决策。Vibe Finance 当前研究的是中国大陆股票和基金的虚拟二级市场组合，不属于上述交易顾问场景。

即使插件内包含 DCF、可比公司或公司简介能力，也不得借此绕过本项目的来源等级、点时证据和策略门禁。只有未来出现明确的并购、发行、债务融资或交易尽调子项目时，才在独立任务中重新评估，不污染当前交易闭环。

## Default templates

状态：**有限接入，仅用于周复盘的派生展示**。

项目当前权威产物是 JSON/JSONL、Markdown 报告和 Python 流水线，没有 Word、Excel 或 PowerPoint 文件。模板不会提高信号预测能力，但 `Analytics Dashboard` 可以提高周度账本的可读性，因此只在周日长周期复盘完成并通过校验后，尝试生成：

```text
artifacts/weekly-dashboard/YYYY-Www-vibe-finance.xlsx
```

仪表盘至少展示项目权益、现金、持仓市值、累计费用、收益率、回撤、成交次数、资产桶暴露和基准对比。所有数值必须由同一周的账本和不可变报告生成，不得补造模板字段。

约束：

- Excel 只是派生展示，`data/ledger/portfolio.json`、`data/ledger/orders.jsonl` 和不可变报告仍是唯一权威记录。
- 模板或 Spreadsheets 能力不可用时，记录 `TEMPLATE_EXPORT_SKIPPED`，周复盘本身不得失败。
- 不在每日盘前、开盘结算、收盘决策或基金净值任务中生成 Office 文件，避免增加延迟和二进制文件噪声。
- 不自动生成投委会备忘录或市场趋势演示文稿；只有用户明确要求时才创建。
- 模板产物同样禁止包含密钥、政治内容或真实投资建议。
