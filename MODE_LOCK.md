# Vibe Finance — MODE LOCK

## 0. 会话与范围

- 生成时间：2026-07-19（Asia/Shanghai）
- 最近治理复核：2026-07-21 12:33（Asia/Shanghai）
- 项目：Vibe Finance
- 目标：构建仅面向中国大陆可获取信息的股票/公募基金虚拟投资研究闭环。
- 语言：中文；仓库文档可保留必要英文标识符。
- Web：确认后允许联网检索；优先监管机构、交易所、基金公司、指数公司、官方披露和可追溯原始数据。
- 执行方式：完整实施，不降级为仅方案或一次性荐股文本。
- 状态标签：事实、推断、假设、UNKNOWN 必须分开。
- 所有交易均为模拟；禁止连接券商、支付或真实下单接口。

## 1. 资金与交易口径

- 初始虚拟本金：30,000.00 元人民币。
- 资金处置授权：可在 0–30,000.00 元之间自主配置；首日允许使用全部本金，也允许保留任意比例现金。
- 不要求满仓；正常交易日必须尝试至少一笔小额虚拟交易。盘前状态未知可生成条件订单，只有确认停牌、双源开盘价失败、价格冲突或系统故障才允许以明确失败结束，禁止伪造成交。
- 研究域：A 股、场内基金、可由中国大陆投资者正常查询的中国公募基金；默认排除港股、美股、期货、期权、融资融券、杠杆/反向产品、私募和加密资产。
- 成交模拟：决策时不得使用未来数据；股票与场内 ETF 的收盘信号按下一交易日双源开盘价模拟，场外基金按未知价原则先登记 `PENDING_NEXT_NAV`、再由信号日之后公开的确认净值结算，同时记录手续费、申赎费和无法成交原因。
- 不允许事后改写历史订单、行情快照或决策理由；更正必须追加记录。

## 2. 数据与来源合同

- 建立来源注册表，至少记录：来源主体、URL、类型、更新频率、权威等级、访问方式、许可/使用边界、失败回退、最后成功时间。
- 一级来源优先：证监会、上交所、深交所、北交所、人民银行、国家统计局、财政部、国家外汇管理局、中国证券投资基金业协会、基金公司公告、上市公司公告、指数公司。
- 二级来源仅用于交叉验证和发现线索：大型财经媒体、券商/资管研究、基金销售平台与聚合行情网站。
- “专家观点”必须保留作者、机构、发布日期、原始链接和利益冲突提示；不得将名气、单条观点或平台热度当作交易信号。
- 所有数值写入 `as_of`、抓取时间、来源和质量标记；数据陈旧、缺失、复权口径不明或交叉验证冲突时，阻止自动调仓。
- 遵守网站服务条款、robots、频率限制和版权要求；不绕过登录、验证码、反爬或付费墙。

## 3. 每日闭环

每次定时运行必须按固定顺序执行：

1. 抓取并校验市场、基金净值、公告、宏观、政策和专家/机构观点。
2. 生成不可变数据快照与来源清单。
3. 更新持仓估值，并对上一决策做归因：市场暴露、行业/风格、个券/基金选择、费用、现金拖累。
4. 计算候选资产的趋势、波动、回撤、流动性、估值/基本面可得指标和相关性；避免只凭叙事。
5. 形成支持与反对两套论证，列出失效条件、最大可接受损失和退出规则。
6. 输出 BUY / ADD / HOLD / REDUCE / SELL / WATCH / NO-TRADE 之一；每个动作必须包含证据时间点、目标金额、成交假设和风险预算。
7. 仅在数据与风险门禁通过时更新虚拟账本；否则保留现金并写明阻断原因。
8. 保存日报、机器可读订单、组合快照、证据清单和运行日志。

## 4. 自进化边界

- 可进化对象：来源可靠性评分、特征权重、候选筛选阈值、仓位约束和分析模板。
- 不可进化对象：模拟交易性质、30,000 元初始本金、禁止未来数据、证据留痕、最大回撤保护、历史不可篡改和真实交易禁令。
- 策略修改必须基于已结束样本，先写假设和评估指标，再同时通过走前与独立样本外比较；不得根据单日盈亏追涨杀跌式改规则。
- 基准至少包含现金、宽基指数/ETF 的可复现对照；绩效至少报告总收益、年化口径（样本足够时）、最大回撤、波动、换手、费用和相对基准表现。
- 每次规则升级都要版本化，并保留旧规则、变更理由、预期改善、反证条件和回滚点。
- 调度分层：交易日收盘后执行短周期；每周执行长周期；高频周期默认禁用，只有点时量化数据、回测、成交/滑点模型和样本外门禁全部通过后才可创建与启用。
- 当前实际调度共 9 个 ACTIVE 任务：每 6 小时活动监测；工作日 08:00 盘前决策、09:10 订单就绪检查、09:35 开盘结算、16:30 收盘分析、22:30 基金净值分析；周六 20:30 反思进化；周日 20:00 长周期复盘；每天 23:10 本地文档日志整理。
- 文档整理任务不得更新金融心跳、账本或订单；反思进化至少需要 20 笔已完成虚拟交易，并通过走前和独立样本外门禁后才可升级策略参数。
- “已完成虚拟交易”只能由合法 `PENDING_NEXT_* → FILLED` 状态机重放得到：成交必须保持订单的 symbol/side/quantity、采用带时区且不早于 signal 的 `fill_as_of`、并且 `simulation_only=true`；仓位从零变正后，只有精确回到零才计一个 round trip。只有进一步携带哈希绑定 provenance 的闭环才进入 eligible sample count；`portfolio.json` 中的可变累计计数不拥有验收权。
- 进化结论只能由受保护的静态 verifier 计算。反思任务只能新增 `reports/evolution/<run-id>/` proposal/candidate/evidence，永久禁止修改 `MODE_LOCK*`、主 Prompt、同步脚本、代码、测试、文档、账本和 live `config/strategy.json`，也不能通过环境变量提交 `ACCEPTED` 或回滚基线。
- 旧订单前四行由 Git blob 与 canonical hash 锚定且不得重写；后续订单事件必须包含连续序号、唯一 event ID、前驱 hash 和事件 hash，且当前工作账本必须以 `HEAD:data/ledger/orders.jsonl` 为严格 canonical 前缀。任何断链、已提交后缀改写、重复/越级成交或组合投影不一致均失败关闭。当前 portfolio 与事件批次尚未形成跨文件原子事务，因此在该缺口关闭前不得声称崩溃一致性。
- 2026-07-21 的 v0.3.0 历史 `ACCEPTED` 标签已由机器门禁判为 `INVALID_LEGACY_ACCEPTANCE`；当前结论是 `PROPOSED_ONLY`。这不自动回滚 live simulation 策略，是否回滚仍需单独决策。
- `Default templates` 仅允许在周复盘后生成派生 Excel 仪表盘；模板文件不参与信号、订单或绩效计算。`Investment Banking` 不得用于当前公开股票/基金决策。
- 策略 v0.3.0 将“正常交易日至少一笔虚拟成交”设为执行目标。常规信号不足时，只能使用通过双源收盘价和公司行为门禁的场内 ETF 小仓探索；盘前状态未知时生成条件订单，09:35 必须验证 `TRADING` 和双源开盘价。休市、停牌、价格冲突或系统故障必须报告失败，不得虚构成交。
- 公开仓库不保存政治或社会议题分析；所有公开页面和报告必须明确说明这是虚拟实验，不构成投资建议。

## 5. 风险硬门禁

- 不做收益保证，不把模拟结果外推为真实收益能力。
- 禁止用未经核验的实时价格成交；禁止幸存者偏差、前视偏差和基金净值发布日期错配。
- 单一股票、单一基金、单一行业和整体权益暴露必须有显式上限；具体数值在首轮数据分布检查后写入版本化策略，变更需留痕。
- 个股需检查 ST/退市风险、停牌、涨跌停、上市天数、流动性、重大公告；基金需检查申赎状态、净值日期、费率、规模、跟踪误差/经理变更和风格漂移。
- 数据不足、市场休市或候选不满足最低证据标准时，默认 NO-TRADE。

## 8. 变更与停止规则

- 范围变更需明确提出并确认，不在日常复盘中悄悄改变初始本金、市场或风险边界；现金与风险资产比例属于策略决策，可随证据调整并留痕。
- 若外部数据的合法稳定接口不可得，停止自动交易更新，退回“生成待审核日报 + NO-TRADE”，而不是伪造连续性。
- 若任务仅部分完成，必须标记 `PARTIALLY_COMPLETED` 并列出剩余工作与下一可执行步骤。
